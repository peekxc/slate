    Introduction

    pICP is an AFIT project designed to create a parallel implementation of an Iterative Closest Point (ICP) algorithm using the CUDA architecture, hence the name “pICP” (parallel Iterative Closest Points). The primary purpose of the project is to produce fast positioning estimates using the well-known ICP algorithm.
    Based on the paper: Besl, Paul J., and Neil D. McKay. “Method for registration of 3-D shapes.” Robotics-DL tentative. International Society for Optics and Photonics, 1992.`

    Background

    The initial alignment phase of ICP is often dominated by the nearest-neighbor computation, also sometimes called the and because recursive spatial-indexing structures such as a full k-D Tree searches generally exhibit low amounts of parallelism, an new approach to the Nearest Neighbor problem is used in this implementation that is conceptually based off the Jump-and-Walk algorithm. Additionally, a parallel point-to-surface projection algorithm that forgoes the need for an expensive square root operation, is also included.

    Requirements

    pICP requires CUDA to run (5.5+). The current API available also requires several C++11 features, however C11 is in the works.

    Getting Started

    pICP leverages both the low-dimensionality of the problem (3) and the abundance of information known at compile-time to achieve its speed estimates. As a result, a lot of preprocessing to get started.

    Note that this implementation is model-specialized and runtime agnostic. This means that, initially, specific model information is required initially at runtime to be loaded onto the GPU. After this is done, any source clouds can be introduced and the algorithm proceeds with the positional estimates (fast), but any reloading of different model data (a different reference model) will require potentially thousands of memory transactions.
